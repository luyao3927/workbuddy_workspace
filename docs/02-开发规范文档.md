## 二、组件开发规范

### 2.1 组件拆分原则

1. **单一职责**：每个组件只负责一个明确的功能
2. **可复用性**：通用组件不耦合业务逻辑，业务组件可跨页面复用
3. **粒度适中**：组件不宜过大（建议 < 300 行）或过小（建议 > 30 行）
4. **层级清晰**：common（通用） > 业务域（dish/cart/category/order） > 页面

### 2.2 组件结构模板

```vue
<template>
  <view class="component-name">
    <!-- 模板内容 -->
  </view>
</template>

<script setup>
// 1. 导入依赖
import { ref, computed, onMounted } from 'vue'
import { useXxxStore } from '@/stores/useXxxStore'

// 2. Props 定义
const props = defineProps({
  itemId: { type: String, required: true },
  isActive: { type: Boolean, default: false },
})

// 3. Emits 定义
const emit = defineEmits(['update', 'delete'])

// 4. Store 引用
const xxxStore = useXxxStore()

// 5. 响应式数据
const localState = ref('')

// 6. 计算属性
const computedValue = computed(() => { /* ... */ })

// 7. 方法
const handleClick = () => {
  emit('update', { id: props.itemId })
}

// 8. 生命周期
onMounted(() => { /* ... */ })
</script>

<style lang="scss" scoped>
.component-name {
  // 样式
}
</style>
```

### 2.3 Props 规范

| 规则           | 说明                                   |
| -------------- | -------------------------------------- |
| 类型声明       | 必须使用 `type` 声明类型                |
| 必填标记       | 关键数据使用 `required: true`           |
| 默认值         | 非必填属性提供合理的 `default` 值       |
| 校验函数       | 复杂校验使用 `validator` 函数           |
| 避免对象默认值 | 对象/数组默认值使用工厂函数返回         |

```javascript
// 正确示例
const props = defineProps({
  dishId: { type: String, required: true },
  quantity: { type: Number, default: 1 },
  tags: {
    type: Array,
    default: () => [],
  },
  size: {
    type: String,
    default: 'medium',
    validator: (value) => ['small', 'medium', 'large'].includes(value),
  },
})
```

### 2.4 Events 规范

| 规则           | 说明                                   |
| -------------- | -------------------------------------- |
| 命名格式       | kebab-case，动词-名词结构               |
| 事件数据       | 使用对象格式传递，便于扩展             |
| 文档注释       | 关键事件添加 JSDoc 注释                |

## 三、样式规范

### 3.1 CSS 变量使用

所有主题色必须通过 CSS 变量引用，严禁硬编码颜色值。

```scss
// 正确 ✅
.dish-card {
  background: var(--bg-primary);
  color: var(--text-primary);
  border: 1px solid var(--border-color);
}

// 错误 ❌
.dish-card {
  background: #FFFFFF;
  color: #333333;
}
```


### 3.2 暗黑模式适配

暗黑模式通过 CSS 变量切换实现，所有组件无需单独编写暗黑样式。

```scss
// theme-light.scss
:root {
  --bg-primary: #FFF8F0;
  --text-primary: #2C3E50;
  --color-primary: #FF6B6B;
  // ...
}

// theme-dark.scss
[data-theme="dark"] {
  --bg-primary: #1A1A2E;
  --text-primary: #ECEFF1;
  --color-primary: #FF8A80;
  // ...
}
```

#### 适配原则

1. **无感知切换**：组件不判断当前主题，仅依赖 CSS 变量
2. **图片适配**：暗黑模式下图片可能需要降低亮度，使用 `filter: brightness(0.8)`
3. **阴影调整**：暗黑模式下阴影应使用更深的颜色
4. **过渡动画**：主题切换添加 `transition: background-color 0.3s, color 0.3s`

### 3.3 手帐风格设计指南

#### 设计原则

- **纸质肌理**：背景模拟纸张质感，使用暖色调（米色/牛皮纸色）
- **拼贴层次**：元素间通过阴影、边框、胶带等营造层次感
- **手写元素**：标题、标签使用手写风格字体
- **清新色调**：以喜茶风格为参考，使用低饱和度、高明度的配色
- **留白呼吸**：适当留白，避免拥挤感

#### 装饰元素使用规范

| 装饰元素       | 使用场景                       | 组件                |

## 七、状态管理规范（Pinia Store）

### 7.1 Store 命名与组织

| Store 文件         | 导出名称          | 职责                           |
| ------------------ | ----------------- | ------------------------------ |
| useAppStore.js     | useAppStore       | 应用全局状态（主题、门店）     |
| useCategoryStore.js | useCategoryStore | 分类数据管理                   |
| useDishStore.js    | useDishStore      | 菜品数据管理                   |
| useCartStore.js    | useCartStore      | 购物车状态管理                 |
| useOrderStore.js   | useOrderStore     | 订单状态管理                   |

### 7.2 Store 结构模板

```javascript
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useXxxStore = defineStore('xxx', () => {
  // ===== 状态 =====
  const items = ref([])
  const loading = ref(false)
  const error = ref(null)

  // ===== 计算属性 =====
  const itemCount = computed(() => items.value.length)

  // ===== 方法 =====
  /**
   * 获取数据
   */
  async function fetchData() {
    loading.value = true
    error.value = null
    try {
      const res = await apiCall()
      items.value = res.data
    } catch (e) {
      error.value = e.message
    } finally {
      loading.value = false
    }
  }

  /**
   * 重置状态
   */
  function reset() {
    items.value = []
    loading.value = false
    error.value = null
  }

  return {
    // 状态
    items,
    loading,
    error,
    // 计算属性
    itemCount,
    // 方法
    fetchData,
    reset,
  }
})
```

### 7.3 Store 使用规范

| 规则               | 说明                                     |
| ------------------ | ---------------------------------------- |
| 状态修改           | 仅通过 Store 暴露的方法修改状态           |
| 异步操作           | 统一使用 async/await，管理 loading/error  |
| 错误处理           | 在 Store 内部 catch，暴露 error 状态      |
| 状态重置           | 每个 Store 提供 `reset()` 方法           |
| 跨 Store 引用      | 允许在 Store 内部引用其他 Store          |
| 持久化             | 购物车数据使用 `storage.js` 持久化       |

### 7.4 购物车持久化示例

```javascript
import { defineStore } from 'pinia'
import { ref, computed, watch } from 'vue'
import { storage } from '@/utils/storage'

export const useCartStore = defineStore('cart', () => {
  const CART_KEY = 'cart_items'

  // 从本地存储恢复
  const items = ref(storage.get(CART_KEY, []))

  // 监听变化自动持久化
  watch(items, (newItems) => {
    storage.set(CART_KEY, newItems)
  }, { deep: true })

  // ... 其他逻辑
})
```

## 八、代码质量要求

### 8.1 通用原则

1. **DRY**：避免重复代码，提取公共逻辑到 composable 或 utils
2. **KISS**：保持代码简单直观，避免过度设计
3. **防御性编程**：对外部数据（API 响应、Props）进行校验和兜底
4. **早返回**：优先处理异常情况，减少嵌套层级

### 8.2 性能优化

1. 列表渲染使用 `v-for` 配合唯一 `key`
2. 大列表考虑虚拟滚动或分页加载
3. 图片使用懒加载（app-image 组件已封装）
4. 避免在模板中使用复杂表达式，使用计算属性替代
5. 合理使用 `v-if` 和 `v-show`（频繁切换用 v-show）

### 8.3 错误处理

```javascript
// 统一错误处理模式
async function fetchData() {
  loading.value = true
  error.value = null
  try {
    const res = await apiCall()
    if (res.code !== 0) {
      throw new Error(res.message || '请求失败')
    }
    items.value = res.data
  } catch (e) {
    error.value = e.message
    console.error('[useXxxStore] fetchData error:', e)
  } finally {
    loading.value = false
  }
}
```
