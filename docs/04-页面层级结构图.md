## 三、数据传递方式汇总

| 传递场景                     | 传递方式            | 说明                                       |
| ---------------------------- | ------------------- | ------------------------------------------ |
| 首页 → 详情页 (dishId)       | URL 参数            | navigateTo 时通过 URL query 传递 dishId    |
| 详情页 ← API (菜品数据)      | Store + API         | useDishStore.fetchDishDetail(dishId)       |
| 详情页 → 购物车 (加购)       | Store 直接调用      | useCartStore.addToCart(item)               |
| 购物车 → 确认订单            | Store 共享状态      | 确认订单页读取 useCartStore 数据            |
| 确认订单 → API (提交)        | Store + API         | useOrderStore.submitOrder(data)            |
| 确认订单 → 下单成功          | Store 共享状态      | 下单成功页读取 useOrderStore 数据           |
| 全局 → 主题切换              | Store + CSS 变量    | useAppStore 切换主题，CSS 变量响应式更新    |
| 全局 → 门店信息              | Config 注入         | app.config.js 配置通过 useAppStore 暴露     |
| 购物车 → 本地持久化          | Storage             | useCartStore 自动 watch 并持久化到本地存储  |
| 页面状态 (加载/空/错误)      | Composable          | usePageState 统一管理各页面状态             |

## 六、页面生命周期与数据加载

```
菜谱首页 onLoad
  │
  ├── useCategoryStore.fetchCategories()
  │     └── API: GET /categories
  │
  └── useDishStore.fetchRecommendedDishes()
        └── API: GET /dishes/recommended

分类切换 (点击 CategoryNav)
  │
  └── useDishStore.fetchDishes({ categoryId, page: 1 })
        └── API: GET /dishes?categoryId=xxx&page=1

菜品详情 onLoad
  │
  └── useDishStore.fetchDishDetail(dishId)
        └── API: GET /dishes/:id

确认订单 onShow
  │
  └── useCartStore (已有数据，直接读取)
        └── 无需 API 调用

下单成功 onLoad
  │
  └── useOrderStore (已有数据，从上一页传递)
        └── 无需 API 调用
```
