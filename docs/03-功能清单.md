## 十、优先级说明

| 优先级 | 含义       | 说明                                       |
| ------ | ---------- | ------------------------------------------ |
| P0     | 必须实现   | 核心功能，MVP 最小可行产品必须包含         |
| P1     | 应该实现   | 重要但非阻塞，建议在首个版本中完成         |
| P2     | 可以延后   | 有价值但可后续迭代，资源允许时实现         |
| P3     | 暂不开发   | 超出当前版本范围，由其他系统或后续版本承接 |
